# A1 Vision Robot Stack 仓库分析文档

## 1. 仓库概览

A1 Vision Robot Stack 是一个基于飞凌微 A1 开发套件与思特威图像传感器的智能小车项目。该项目旨在实现以下核心功能：

- YOLOv8 训练与部署（ONNX）
- 手势识别、目标跟踪、视觉检测
- 避障与底盘控制（GPIO、串口）
- 深度感知与环境点云生成
- 前端视频输出与可视化

## 2. 目录结构与代码放置逻辑

### 2.1 整体目录结构

```text
.
├── .github/             # GitHub CI/CD 配置
├── data/                # 训练数据模板和SDK
│   └── yolov8_dataset/  # YOLOv8 数据集结构
├── docker/              # 容器构建与运行配置
├── docs/                # 项目文档
├── src/                 # 源代码目录
│   ├── a1_ssne_ai_demo/ # 官方 demo 同步副本
│   └── ros2_ws/         # ROS2 工作区
│       └── src/a1_robot_stack/  # 主要功能包
├── tools/               # 工具脚本
│   └── yolov8/          # YOLOv8 相关工具

```

### 2.2 代码放置逻辑

1. **ROS2 工作区结构**：
   - 遵循标准 ROS2 工作区结构，所有功能代码位于 `src/ros2_ws/src/a1_robot_stack/` 目录
   - 节点代码直接放在 `src/` 目录下
   - 模块代码放在 `src/modules/` 目录下
   - 头文件放在 `include/a1_robot_stack/` 目录下
   - 启动文件放在 `launch/` 目录下

2. **功能分离原则**：
   - 每个节点负责单一功能，通过 ROS2 话题进行通信
   - 感知、控制、安全监管等功能清晰分离
   - 模块设计遵循高内聚、低耦合原则

3. **硬件接口抽象**：
   - 通过参数配置支持不同硬件接入方式
   - 提供模拟输入模式，便于无硬件时的开发调试

## 3. 核心功能模块详解

### 3.1 感知节点 (perception_node.cpp)

**功能**：视觉感知主循环，处理摄像头输入并进行推理

**核心逻辑**：
- 支持两种输入模式：模拟输入和真实相机输入
- 模拟输入模式：生成伪深度数据和手势信息
- 真实相机输入：订阅相机话题，计算灰度均值作为深度代理
- 发布导航命令、障碍物检测结果、深度信息和手势识别结果
- 实现相机超时监控和故障上报

**关键参数**：
- `target_fps`：目标帧率
- `model_path`：模型路径
- `use_mock_input`：是否使用模拟输入
- `camera_topic`：相机话题名
- `camera_timeout_sec`：相机超时阈值

### 3.2 激光雷达接入节点 (lidar_ingest_node.cpp)

**功能**：处理激光雷达数据，检测近距离障碍物

**核心逻辑**：
- 支持三种雷达接入模式：话题模式、串口模式和RPLidar SDK模式
- 话题模式：订阅 `/scan` 话题，计算最近障碍物距离
- 串口模式：直接从串口读取雷达数据
- RPLidar SDK模式：使用RPLidar SDK直接驱动Slamtec激光雷达
- 发布最近障碍物距离和障碍物检测结果
- 实现故障上报机制

**关键参数**：
- `obstacle_threshold_m`：障碍物阈值
- `use_scan_topic`：是否使用话题模式
- `use_rplidar_sdk`：是否使用RPLidar SDK模式
- `scan_topic`：雷达话题名
- `serial_port`：串口设备路径
- `serial_baud`：串口波特率
- `serial_poll_ms`：串口轮询间隔

### 3.3 底盘控制节点 (chassis_controller_node.cpp)

**功能**：接收安全命令并控制底盘运动

**核心逻辑**：
- 订阅安全命令话题 `/cmd_vel_safe`
- 限制线速度和角速度在安全范围内
- 支持 UART 输出控制底盘
- 发布实际输出命令和心跳信息

**关键参数**：
- `max_linear`：最大线速度
- `max_angular`：最大角速度
- `use_uart_output`：是否使用 UART 输出
- `uart_port`：UART 端口路径
- `uart_baud`：UART 波特率

### 3.4 安全监管节点 (safety_supervisor_node.cpp)

**功能**：监控系统各组件心跳，处理故障，实现紧急停车

**核心逻辑**：
- 订阅导航命令和故障信息
- 监控视觉、雷达、底盘、显示四个节点的心跳
- 当检测到心跳超时或严重故障时，触发紧急停车
- 发布安全命令和告警信息

**关键参数**：
- `heartbeat_timeout_sec`：心跳超时阈值

### 3.5 显示桥接节点 (display_bridge_node.cpp)

**功能**：汇总感知和告警信息，输出显示文本

**核心逻辑**：
- 订阅深度、障碍物、手势和告警信息
- 定期发布叠加文本信息
- 将状态写入文件，供 HDMI 控制台查看
- 发布心跳信息

**关键参数**：
- `status_file_path`：状态文件路径
- `publish_period_ms`：发布周期

### 3.6 性能监控节点 (performance_monitor_node.cpp)

**功能**：监控系统性能，包括帧率、丢帧率和波动

**核心逻辑**：
- 订阅视觉帧率信息
- 计算平均帧率、95% 分位数帧率、丢帧率和波动
- 当性能下降时发布警告

**关键参数**：
- `target_fps`：目标帧率

## 5. 后续开发建议

### 5.1 必须完成的集成点

1. **A1 摄像头灰度流接入**：替换当前的模拟输入，实现真实相机数据的处理
2. **YOLO ONNX 推理接入**：集成 A1 NPU 或板端可用的推理后端
3. **RPLidar SDK 集成**：完善 RPLidar SDK 的使用，实现激光雷达数据的直接获取和处理
4. **雷达 + 视觉深度融合**：结合雷达和视觉数据，生成真实的 PointCloud2
5. **底盘串口协议适配**：实现速度命令的下发与回读
6. **场景鲁棒性策略**：实现曝光/增益/去噪的自适应调整
7. **功耗与实时性优化**：动态调频、线程绑核、零拷贝等优化

### 5.2 模块扩展建议

1. **深度感知模块**：实现基于单目或双目相机的深度估计
2. **目标跟踪模块**：实现多目标跟踪功能
3. **路径规划模块**：基于感知信息实现路径规划
4. **SLAM 模块**：实现同步定位与地图构建
5. **远程控制模块**：添加远程控制接口

### 5.3 测试与验证建议

1. **单元测试**：为每个模块编写单元测试
2. **集成测试**：测试节点间的交互
3. **性能测试**：评估系统在不同负载下的表现
4. **鲁棒性测试**：测试系统在各种环境下的稳定性
5. **实车测试**：在实际硬件上验证系统功能

## 6. 开发流程指南

### 6.1 环境准备

1. **硬件准备**：飞凌微 A1 开发板、思特威 SC132GS 相机、激光雷达、底盘
2. **软件环境**：
   - Windows + Docker Desktop
   - Git
   - Python 3.9（用于 YOLOv8 训练）

### 6.2 开发步骤

1. **克隆代码**：
   ```powershell
   git clone https://github.com/GitLaughs/See-you-more-than-her.git
   cd See-you-more-than-her
   ```

2. **启动开发容器**：
   ```powershell
   docker compose -f docker/docker-compose.yml up -d
   ```

3. **编译 SDK**：
   ```powershell
   docker exec A1_Builder bash -lc "cd /app/smartsens_sdk/A1_SDK_SC132GS/smartsens_sdk; make BR2_EXTERNAL=./smart_software smartsens_m1pro_release_defconfig"
   docker exec A1_Builder bash -lc "cd /app/smartsens_sdk/A1_SDK_SC132GS/smartsens_sdk; make m1_sdk_lib-rebuild"
   ```

4. **编译 ROS2**：
   ```powershell
   docker exec A1_Builder bash -lc "cd /app/src/ros2_ws; source /opt/ros/jazzy/setup.bash; colcon build --symlink-install"
   ```

5. **启动系统**：
   ```powershell
   docker exec A1_Builder bash -lc "cd /app/src/ros2_ws; source /opt/ros/jazzy/setup.bash; source install/setup.bash; ros2 launch a1_robot_stack bringup.launch.py"
   ```

6. **低负载核心启动**（推荐板端联调）：
   ```powershell
   docker exec A1_Builder bash -lc "cd /app/src/ros2_ws; source /opt/ros/jazzy/setup.bash; source install/setup.bash; ros2 launch a1_robot_stack bringup_a1_core.launch.py"
   ```

### 6.3 代码提交流程

1. **每天开工前**：拉取最新代码
   ```powershell
   git checkout main
   git pull --rebase origin main
   ```

2. **创建功能分支**：
   ```powershell
   git checkout -b feat/你的功能名
   ```

3. **提交代码**：
   ```powershell
   git status
   git diff
   git add 文件1 文件2
   git commit -m "feat: 本次改动说明"
   git push origin 你的分支名
   ```

4. **发起 PR**：在 GitHub 上发起 Pull Request，描述改动内容、验证方法和影响范围

## 7. 总结

A1 Vision Robot Stack 是一个基于飞凌微 A1 开发套件的智能小车项目，通过 ROS2 实现了感知、控制、安全监管等核心功能。项目采用模块化设计，各节点职责明确，通过话题进行通信，具有良好的可扩展性。

后续开发需要完成相机接入、YOLO 推理、深度融合等关键集成点，同时可以扩展更多功能模块，如路径规划、SLAM 等。通过严格的测试和验证流程，确保系统在实际应用中的稳定性和可靠性。

项目的成功实现将为智能小车的开发提供一个完整的参考方案，展示如何结合视觉、雷达等传感器实现自主导航和避障功能。
