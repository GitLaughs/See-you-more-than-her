# 4. 数据集与数据处理

## 4.1 数据来源

训练数据来自 `data/rps_dataset/` 下采集的视频素材。原始数据按类别组织，覆盖 `person`、`stop`、`forward`、`obstacle` 和 `NoTarget` 等状态。视频内容包含不同拍摄角度、不同距离和不同光照条件下的目标状态，用于支撑五分类模型训练。

从工程角度看，当前项目并不直接使用视频进行训练，而是先将视频统一转换为裁剪后的图片数据集，再送入训练脚本。这样做的好处是数据格式更稳定、训练输入更统一，也更方便后续导出和板端部署时保持一致。

## 4.2 类别设计

五分类任务的类别定义如下：

| 类别 | 含义 | 用途 |
| --- | --- | --- |
| `person` | 人物或人体目标 | 人物状态识别 |
| `stop` | 停止手势或停止状态 | 触发停止语义 |
| `forward` | 前进手势或前进状态 | 触发前进语义 |
| `obstacle` | 障碍物状态 | 用于避障或告警语义 |
| `NoTarget` | 无有效目标 | 空场景、待机或兜底状态 |

这套类别定义的核心目标不是扩展通用识别能力，而是让模型输出能够直接映射到项目中的动作语义。`NoTarget` 的加入也很重要，它保证了空场景下系统仍然有明确输出，避免板端在无目标时产生不稳定控制。

## 4.3 数据预处理流程

当前数据处理流程已经固定为“视频 → 抽帧 → 裁剪 → 灰度化 → 分割数据集”五个步骤：

1. 从原始视频读取帧；
2. 按固定帧间隔抽样，减少连续帧冗余；
3. 对画面做中心裁剪，再截取固定区域；
4. 将裁剪结果转为灰度图并保存为图片；
5. 按类别和帧数划分为 `train / val / test`。

对应处理脚本为 `demo-rps/dataprocess_modeltrain/prepare_video_dataset.py`。该脚本默认从 `data/rps_dataset/datasets/` 读取视频，输出到 `data/rps_dataset/processed_dataset/` 或当前使用的 `data/rps_dataset/processed_dataset_regularized/`。

脚本关键参数如下：

- `--dataset_dir`：原始视频目录；
- `--output_dir`：处理后数据集目录；
- `--frame_step`：抽帧间隔，默认 `3`；
- `--train_ratio`：训练集比例，默认 `0.8`；
- `--val_ratio`：验证集比例，默认 `0.1`；
- `--crop`：固定裁剪区域，默认 `0 0 320 320`；
- `--image_ext`：输出图片格式，默认 `.png`。

## 4.4 处理后数据集结构

当前稳定使用的数据集目录为：

```text
data/rps_dataset/processed_dataset_regularized/
  train/
    person/
    stop/
    forward/
    obstacle/
    NoTarget/
  val/
    person/
    stop/
    forward/
    obstacle/
    NoTarget/
  test/
    person/
    stop/
    forward/
    obstacle/
    NoTarget/
```

这一目录结构和训练脚本输入要求完全一致。训练脚本会直接读取 `train`、`val`、`test` 三个 split 下的类别目录，并按照目录名映射到固定标签顺序。

## 4.5 数据划分优化

早期若按视频数量划分，容易出现某些 split 中缺少某一类别的问题，尤其是样本量较少的 `NoTarget` 类。这样一来，训练脚本在启动时就可能因为某个 split 缺类而直接失败，影响中期演示和反复训练效率。

当前方案改为按抽样帧数量划分，而不是单纯按视频数量划分。这样处理后，train、val、test 更容易同时覆盖全部类别，训练稳定性更高，也更适合小样本、固定类别的工程任务。

## 4.6 当前数据状态

当前项目已经形成较稳定的数据准备结果，包括：

- 原始视频数据集；
- 处理后的帧图片数据集；
- 平衡后的训练数据；
- 训练输出权重和元数据；
- 可继续用于后续训练和导出的标准化目录。

这说明项目的数据链路已经从“能采集”发展到“能稳定训练”，为后续模型训练、评估和部署提供了可复现基础。
