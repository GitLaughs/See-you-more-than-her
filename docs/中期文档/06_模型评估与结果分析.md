# 6. 模型评估与结果分析

## 6.1 核心评估指标

本项目评估指标覆盖整体准确率、训练稳定性和类别均衡性三个层面。训练脚本记录的核心指标包括：

- `train_loss`；
- `val_loss`；
- `train_top1`；
- `val_top1`；
- `test_top1`；
- 每类 `precision`；
- 每类 `recall`；
- 每类 `F1`；
- 混淆矩阵。

其中，`top1` 用于表示模型输出最大 logit 是否对应真实类别，混淆矩阵用于分析类别间误判关系，每类 F1 用于判断各类别识别是否均衡。

## 6.2 评估实现

训练脚本中通过 logits 最大值计算预测类别：

```python
def accuracy_from_logits(logits: torch.Tensor, targets: torch.Tensor):
    preds = logits.argmax(dim=1)
    return (preds == targets).float().mean().item()
```

混淆矩阵以真实类别为行、预测类别为列进行统计：

```python
def confusion_matrix(preds: torch.Tensor, targets: torch.Tensor, num_classes: int):
    matrix = torch.zeros((num_classes, num_classes), dtype=torch.int64)
    for target, pred in zip(targets.tolist(), preds.tolist()):
        matrix[target, pred] += 1
    return matrix
```

每类指标由混淆矩阵计算得到：

```python
precision = tp / (tp + fp) if tp + fp else 0.0
recall = tp / (tp + fn) if tp + fn else 0.0
f1 = 2 * precision * recall / (precision + recall) if precision + recall else 0.0
```

该评估方式能够同时反映整体性能和单类误判情况，适合用于中期报告和答辩展示。

## 6.3 结果文件生成

训练结束后，脚本统一生成 CSV 与图像报告。实现入口如下：

```python
def save_training_reports(output_dir: Path, history: list[dict], val_matrix: torch.Tensor, test_matrix: torch.Tensor, class_names: list[str]):
    save_results_csv(output_dir, history)
    plot_training_curves(output_dir, history)
    plot_confusion_matrix(output_dir / "confusion_matrix_val.png", val_matrix, class_names, "Validation Confusion Matrix")
    plot_confusion_matrix(output_dir / "confusion_matrix_test.png", test_matrix, class_names, "Test Confusion Matrix")
    save_per_class_metrics(output_dir, val_matrix, class_names)
```

其中：

- `results.csv` 用于记录每轮训练损失和 top1；
- `accuracy_loss_curve.png` 用于展示训练与验证趋势；
- `confusion_matrix_val.png` 用于展示验证集类别混淆情况；
- `confusion_matrix_test.png` 用于展示测试集类别混淆情况；
- `per_class_metrics.csv` 用于记录每类 precision、recall、F1 和 support。

## 6.4 分析重点

中期阶段模型评估不只关注最终准确率，还需要分析以下内容：

1. 训练集和验证集 loss 是否同步下降；
2. 验证集 top1 是否稳定；
3. 是否出现训练集提升但验证集停滞的过拟合现象；
4. `forward` 与 `stop` 是否存在明显混淆；
5. `NoTarget` 是否被误判为有效目标；
6. 每类 F1 是否均衡。

其中，`forward` 和 `stop` 的混淆会直接影响底盘动作，`NoTarget` 的误判会影响空场景安全性。因此，上述类别需要作为后续数据补充和训练优化重点。

## 6.5 中期展示材料

中期报告和 PPT 中建议展示以下材料：

- 训练曲线；
- 验证集混淆矩阵；
- 测试集混淆矩阵；
- 每类 F1 指标；
- YOLO 检测方案切换为五分类方案的原因；
- 五分类模型适合 A1 板端部署的依据。

该展示方式能够覆盖“模型是否能训练”“训练是否稳定”“结果是否可解释”“是否适合部署”四个评审关注点。
