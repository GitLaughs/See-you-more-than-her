# 1. 项目概述

## 1.1 项目背景

本项目面向 SmartSens A1 视觉板与 STM32 智能小车联动场景，构建了一套基于端侧视觉识别的交互控制系统。系统以 A1 视觉板作为图像采集、AI 推理、OSD 显示和 UART 指令输出核心，以 STM32 控制板作为底盘运动执行核心，并通过 Windows 上位机工具完成相机预览、串口调试和联调验证，形成“视觉感知—状态显示—控制执行”一体化链路。

与传统依赖目标检测框的方案相比，当前项目中期主线调整为轻量级五分类识别方案，更适合 A1 板端实时部署与演示验证。五分类方案直接输出状态语义，不需要额外进行检测框坐标还原和复杂后处理，能够缩短推理链路、降低部署难度，并提升调试与联动控制的稳定性。

当前项目的五分类类别定义为 `person`、`stop`、`forward`、`obstacle` 和 `NoTarget`。其中，`person`、`stop`、`forward`、`obstacle` 用于表达项目中的典型视觉状态，`NoTarget` 作为空场景或无有效目标状态参与统一训练与推理。该类别体系既满足中期演示对动作语义的表达需求，也为后续板端控制映射提供了清晰、稳定的输入基础。

## 1.2 项目目标

本项目的目标不是追求通用大模型效果，而是围绕 A1 板端演示链路，构建一个稳定、轻量、可部署、可调试的视觉识别与控制系统。具体目标如下：

1. 实现 A1 端摄像头图像采集与预览；
2. 在 A1 端运行轻量级五分类模型，完成视觉状态识别；
3. 通过 OSD 叠加显示识别结果、系统状态和动作提示；
4. 通过 UART 将视觉识别结果发送给 STM32 控制板；
5. 由 STM32 根据指令控制小车完成前进、停止等动作；
6. 使用 Windows 侧工具辅助摄像头预览、串口调试和联调验证；
7. 形成从数据处理、训练、验证、测试、导出到部署验证的完整工程闭环。

## 1.3 中期阶段重点

中期阶段的工作重点已经从“方案探索”转向“主线收敛”。项目早期曾尝试基于 YOLO 的检测式方案，用于探索目标检测式视觉交互路径；但在当前部署约束和演示目标下，该方案的稳定性与匹配度不足，因此中期阶段将主线调整为轻量五分类模型方案。

围绕这一调整，中期阶段重点完成了以下几项工作：

- 明确系统总体结构和模块边界；
- 完成五分类模型训练链路；
- 梳理板端运行、Windows 调试和 STM32 控制路径；
- 完善训练结果记录、混淆矩阵和每类指标输出；
- 为中期报告、PPT 和演示视频准备可复现、可展示的技术材料。

这一阶段的成果表明，项目已经从单纯的功能探索，收敛到可训练、可导出、可部署、可验证的工程实现阶段。
