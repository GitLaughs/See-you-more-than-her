# 13. 结论与后续优化方向

## 13.1 中期阶段结论

本项目中期阶段已经形成以 SmartSens A1 视觉板为核心、STM32 控制板为执行端、Windows 工具为联调端的嵌入式视觉控制系统。系统当前主线从早期 YOLO 检测方案收敛为轻量五分类模型方案，完成了从数据处理、模型训练、结果评估、ONNX 导出到板端部署准备的完整链路。

从工程状态看，项目已经具备以下成果：

1. 明确了 A1、STM32 和 Windows 工具之间的模块边界；
2. 完成了五分类数据集处理和训练流程；
3. 实现了 MobileNetV1 五分类模型训练；
4. 支持生成权重、元数据、训练曲线、混淆矩阵和每类指标；
5. 支持导出固定输入尺寸的 ONNX 模型；
6. 板端运行逻辑已经对齐五分类输出和控制语义；
7. Windows 工具已经覆盖视觉预览、直连 STM32 和 A1 中继联调。

上述成果说明项目已经具备中期提交所需的核心技术内容和工程材料基础。

## 13.2 当前技术路线价值

当前五分类方案的技术价值主要体现在以下方面：

- 推理链路短，适合 A1 端部署；
- 输出语义明确，便于动作映射；
- 不依赖检测框还原，降低后处理复杂度；
- 类别体系固定，便于训练、导出和板端一致性维护；
- 训练输出完整，便于中期答辩展示和后续实验复现。

该路线符合当前硬件约束和演示目标，更适合作为中期阶段的主展示方案。

## 13.3 后续优化方向

后续优化建议集中在以下方向：

1. **继续补充数据**  
   增加弱光、遮挡、不同距离、不同角度下的样本，提升模型鲁棒性。

2. **细化类别边界**  
   重点观察 `forward` 与 `stop` 的混淆情况，必要时补充针对性样本。

3. **完善板端部署验证**  
   确认训练输入、ONNX 输出、`.m1model` 推理结果和板端类别顺序完全一致。

4. **补充联调证据**  
   保存 Aurora 预览、A1_TEST 回包、STM32 控制响应和 OSD 显示结果，作为答辩材料。

5. **完善性能数据**  
   在报告中补充训练准确率、测试准确率、混淆矩阵和每类 F1 指标。

6. **整理提交材料**  
   按提交要求整理技术报告、PPT、技术数据压缩包和演示视频。

## 13.4 总结

本项目已经完成从方案探索到工程收敛的关键转变。当前系统以五分类模型为核心，围绕 A1 板端部署、STM32 底盘控制和 Windows 联调工具形成完整链路。该成果具备清晰的设计目标、明确的实现路径、可验证的训练输出和后续优化方向，可作为中期作品提交的核心技术内容。
