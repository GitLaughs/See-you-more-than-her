# 03 编译与烧录

本文档按当前脚本整理完整构建、增量构建、ROS2 单独构建和板端部署路径。

## 前提
- `A1_Builder` 容器已启动
- SDK 已通过 `bootstrap.sh` 或手动方式准备好
- 相关入口参见 `docs/02_环境搭建.md`

## 完整固件构建

```bash
docker exec A1_Builder bash -lc "bash /app/scripts/build_complete_evb.sh"
docker exec A1_Builder bash -lc "bash /app/scripts/build_complete_evb.sh --skip-ros"
docker exec A1_Builder bash -lc "bash /app/scripts/build_complete_evb.sh --app-only"
```

可选清理重构：

```bash
docker exec A1_Builder bash -lc "bash /app/scripts/build_complete_evb.sh --clean"
```

## 这条构建链到底产出什么
`scripts/build_complete_evb.sh` 会依次重建 Demo、可选构建 ROS2、再重新打包 SDK 镜像，所以最终可烧录产物是：

```text
output/evb/<timestamp>/zImage.smartsens-m1-evb
```

不是单独的 `ssne_ai_demo`。

## ROS2 单独构建

```bash
bash scripts/build_ros2_ws.sh
bash scripts/build_ros2_ws.sh --clean
bash scripts/build_ros2_ws.sh wheeltec_robot_msg turn_on_wheeltec_robot
```

说明：
- 只扫描 `src/ros2_ws/src/` 下的包
- 一些可选包故意保留 `COLCON_IGNORE`
- 不要把 ROS2 工作区等同于默认板端运行栈

## 增量构建

```bash
bash scripts/build_incremental.sh sdk ssne_ai_demo
bash scripts/build_incremental.sh sdk m1_sdk_lib
bash scripts/build_incremental.sh sdk linux
bash scripts/build_incremental.sh ros wheeltec_multi
```

适用场景：
- 改 `ssne_ai_demo`：先走 `sdk ssne_ai_demo`
- 改 SDK 基础库：走 `sdk m1_sdk_lib`
- 改打包/内核相关：走 `sdk linux`
- 改 ROS2 包：走 `ros <package>`

## 烧录边界
- Aurora Companion 不是固件烧录器
- 文档中的 `tools/aurora/launch.ps1` 只负责 Windows 侧联调
- 固件烧录仍使用官方 `Aurora.exe` 或当前项目采用的板端烧录流程

## 板端运行验证

```bash
ssh root@<A1_IP>
/app_demo/scripts/run.sh
```

如果只是确认镜像里应用是否可跑，这一步最直接。

## 常见构建场景

### 只改板端 Demo
1. `bash scripts/build_incremental.sh sdk ssne_ai_demo`
2. 如需完整可烧录镜像，再跑 `build_complete_evb.sh --app-only` 或完整构建

### 改了 ROS2 包
1. `bash scripts/build_ros2_ws.sh <package-name>`
2. 需要把 ROS2 内容重新带进整包镜像时，再跑 `build_complete_evb.sh`

### 想出一份完整发布镜像
直接跑：

```bash
docker exec A1_Builder bash -lc "bash /app/scripts/build_complete_evb.sh"
```

## 常见问题

### `output/evb/` 没有新固件
确认执行的是 `build_complete_evb.sh`，而不是只做了 `build_incremental.sh`。

### ROS2 包编不过
先确认包在 `src/ros2_ws/src/` 下，且没有 `COLCON_IGNORE`。

### 板端运行正常，但找不到完整烧录产物
去 `output/evb/<timestamp>/` 看 `zImage.smartsens-m1-evb`，不要只盯 `ssne_ai_demo`。
