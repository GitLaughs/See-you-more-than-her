# 项目现状与后续方向

## 当前已落地
- 板端 `ssne_ai_demo` 已具备 demo-rps 风格的 RPS 分类、OSD、A1_TEST、Link-Test、STM32 UART 控制
- `scripts/build_complete_evb.sh` 已打通完整 EVB 打包路径
- `tools/aurora/` 已具备视频预览、A1_TEST 终端、双模式底盘调试

## 当前系统由什么组成
- 板端 Demo：`data/A1_SDK_SC132GS/smartsens_sdk/smart_software/src/app_demo/face_detection/ssne_ai_demo/`
- SDK / 固件打包：`data/A1_SDK_SC132GS/smartsens_sdk/`
- Aurora：`tools/aurora/`
- STM32 集成参考：`src/stm32_akm_driver/`

## 当前边界
- 默认板端运行路径仍是 `ssne_ai_demo`
- 当前评委演示主线是 demo-rps 风格的 RPS 分类测试，不是 YOLO 检测
- `output/` 是构建产物，不是源码事实来源

## 当前协作重点
- 文档、脚本、Aurora 和 `ssne_ai_demo` 属于优先维护区域
- SDK vendor-heavy 区域、外部依赖目录和厂商内容需要谨慎修改
- 功能描述优先以当前代码和脚本为准，不沿用历史计划表述

## 后续方向

### Aurora 稳定性与联调体验持续改进
继续优化相机初始化、串口反馈和 UI 可观测性，减少 Windows 侧联调摩擦。

### 板端 app 与镜像打包链路的迭代验证
围绕 `ssne_ai_demo`、A1_TEST 和 `build_complete_evb.sh` 持续收敛验证路径，确保最终关注的是可烧录镜像而不是单个二进制。
