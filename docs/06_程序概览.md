# 06 程序概览

## 本文定位
帮助第一次读代码的人快速找到板端 Demo、SDK 打包、Aurora 分别在哪，以及改不同问题时应该先进哪一层。

## 四个主要区域
- `data/A1_SDK_SC132GS/smartsens_sdk/smart_software/src/app_demo/face_detection/ssne_ai_demo/`：板端推理、OSD、A1_TEST、UART 控制
- `data/A1_SDK_SC132GS/smartsens_sdk/`：镜像打包和 SDK 构建基础
- `tools/aurora/`、`tools/PC/`、`tools/A1/`：Windows 联调工具
- `src/stm32_akm_driver/`：STM32 集成参考资料

## 代码库怎么分层看

### 板端 Demo
默认上板运行路径仍是 `ssne_ai_demo`。当前主线已经切成 demo-rps 风格的 RPS 分类测试，板端模型路径是 `model_rps.m1model`，不是 YOLO 检测。

### SDK / 固件打包层
这层负责把最新应用和相关内容重新打进 EVB 镜像。最终部署关注的是 `zImage.smartsens-m1-evb`，不是单独的 Demo 二进制。

### Windows 工具
`tools/aurora/` 负责相机预览、拍照、COM13 终端和 A1_TEST 手动测试；`tools/PC/` 负责电脑直连 STM32 调试；`tools/A1/` 负责 COM13 → A1_TEST → STM32 中继控制。三套工具入口、页面和默认端口相互独立。

## 修改任务怎么定位入口
- 改板端行为、A1_TEST、OSD、UART 控制：先看 `ssne_ai_demo/`
- 改完整镜像产物、打包流程、构建脚本：先看 `scripts/` 和 SDK 打包路径
- 改桌面联调、相机预览、串口终端：先看 `tools/aurora/`、`tools/PC/`、`tools/A1/`
- 改 STM32 协议背景或参考资料：看 `src/stm32_akm_driver/`

## 运行入口怎么找
- 板端验证：`cd /app_demo && ./scripts/run.sh`
- 完整镜像构建：`scripts/build_complete_evb.sh`
- Aurora 入口：`tools/aurora/aurora_companion.py`
- PC 工具入口：`tools/PC/pc_tool.py`
- A1 工具入口：`tools/A1/a1_tool.py`
- Windows 启动脚本：`tools/aurora/launch.ps1`、`tools/PC/launch.ps1`、`tools/A1/launch.ps1`
