# 06 程序概览

## 本文定位
帮助第一次读代码的人快速找到板端 Demo、SDK 打包、ROS2、Aurora 分别在哪，以及改不同问题时应该先进哪一层。

## 四个主要区域
- `data/A1_SDK_SC132GS/smartsens_sdk/smart_software/src/app_demo/face_detection/ssne_ai_demo/`：板端推理、OSD、A1_TEST、UART 控制
- `data/A1_SDK_SC132GS/smartsens_sdk/`：镜像打包和 SDK 构建基础
- `src/ros2_ws/`：ROS2 Jazzy 工作区
- `tools/aurora/`：Windows 联调工具

## 代码库怎么分层看

### 板端 Demo
默认上板运行路径仍是 `ssne_ai_demo`。图像采集、SCRFD/YOLOv8 推理、OSD、A1_TEST 和到底盘的 UART 控制都在这层。

### SDK / 固件打包层
这层负责把最新应用和相关内容重新打进 EVB 镜像。最终部署关注的是 `zImage.smartsens-m1-evb`，不是单独的 Demo 二进制。

### ROS2 工作区
`src/ros2_ws/` 是独立工作区和后续集成路径，不等于默认板端运行栈。`scripts/build_ros2_ws.sh` 只扫描 `src/ros2_ws/src/`，部分可选包故意保留 `COLCON_IGNORE`。

### Aurora
`tools/aurora/` 负责 Windows 侧相机预览、A1_TEST 串口调试、直连 STM32 控制和 ROS bridge 联调。UI、Flask 路由、相机桥、串口和 ROS 辅助逻辑都在这里汇合。

## 修改任务怎么定位入口
- 改板端行为、A1_TEST、OSD、UART 控制：先看 `ssne_ai_demo/`
- 改完整镜像产物、打包流程、构建脚本：先看 `scripts/` 和 SDK 打包路径
- 改桌面联调、相机预览、串口终端：先看 `tools/aurora/`
- 改机器人节点、colcon 构建、ROS 包启停：先看 `src/ros2_ws/`
- 改 STM32 协议背景或参考资料：看 `src/stm32_akm_driver/`

## 运行入口怎么找
- 板端验证：`/app_demo/scripts/run.sh`
- 完整镜像构建：`scripts/build_complete_evb.sh`
- ROS2 构建：`scripts/build_ros2_ws.sh`
- Aurora 入口：`tools/aurora/aurora_companion.py`
- Windows 启动脚本：`tools/aurora/launch.ps1`
