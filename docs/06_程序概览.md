## 三、核心处理流程

### 3.1 主循环（`demo_face.cpp`）

```
每帧处理 (main 循环):
  1. IMAGEPROCESSOR::GetImage() → 采集 720×1280 Y8 全分辨率竖屏帧（无裁剪）
  2. 推理后端选择：
     - 若 USE_SCRFD_BACKEND=true → SCRFDGRAY::Predict()
     - 若 USE_SCRFD_BACKEND=false → YOLOV8::Predict()
  3. YOLOV8 推理细节：
     → RunAiPreprocessPipe 硬件缩放至 640×360
     → SSNE NPU 推理 (YOLOv8 head6)
     → DFL softmax decode + sigmoid + NMS (CPU)
     → 坐标 ×2 映射回 720×1280（w_scale=2, h_scale=2）
  4. 决策与底盘控制（优先级：A1_TEST > Link-Test > 检测结果）：
     - 若收到 A1_TEST stop/move → 立即执行，暂停 Link-Test
     - 若 Link-Test 模式启用：
         → 周期前进1秒（60 mm/s）、停车4秒
     - 若 Link-Test 关闭：
         → 检测到目标 → ChassisController::SendVelocity(100, 0, 0)
         → 未检测到目标 → ChassisController::SendVelocity(0, 0, 0)
  5. VISUALIZER::Draw() → OSD 叠加检测框（720×1280 竖屏坐标系）

键盘监听线程：
  - 读取 stdin，处理 help/stop/move/link_test 等命令
  - 线程安全：使用 std::mutex 保护全局变量

A1_TEST 协议处理：
  - 通过 COM13 接收文本命令
  - 解析后执行对应操作
  - 返回 JSON 格式响应
```

### 3.2 A1_TEST 协议命令一览

| 命令 | 说明 | JSON 响应示例 |
| --- | --- | --- |
| `help` | 显示可用命令列表 | `{"success": true, "message": "..."}` |
| `status` | 查询系统状态 | `{"success": true, "sensor_width": 720, ...}` |
| `A1_TEST test_echo <msg>` | 回显测试 | `{"success": true, "echo": "..."}` |
| `A1_TEST debug_status` | 查询调试状态快照 | `{"success": true, "frame_index": N, ...}` |
| `A1_TEST link_test on/off` | 开关 Link-Test 模式 | `{"success": true, "enabled": true}` |
| `A1_TEST stop` | 发送停车指令并暂停 Link-Test | `{"success": true, "message": "已发送停车指令"}` |
| `A1_TEST move <vx> <vy> <vz>` | 手动运动控制（单位：mm/s） | `{"success": true, "vx": 100, ...}` |

### 3.3 Link-Test 模式逻辑

```
Link-Test 周期：5秒
  ├─ 0.0 ~ 1.0秒：前进（vx=60 mm/s）
  └─ 1.0 ~ 5.0秒：停车（vx=0）

可通过 A1_TEST 随时切换或停止
```

## 四、通信协议

### 4.1 A1 → STM32 控制帧（11 字节）

```
字节  内容        说明
[0]   0x7B       帧头
[1]   Cmd        命令: 0x00=正常控制, 0xFF=IP帧
[2]   0x00       保留
[3]   Vx_H       X轴速度高字节 (mm/s, int16_t)
[4]   Vx_L       X轴速度低字节
[5]   Vy_H       Y轴速度高字节 (AKM车=0)
[6]   Vy_L       Y轴速度低字节
[7]   Vz_H       Z轴转向角速度高字节 (AKM车=前轮转角)
[8]   Vz_L       Z轴转向角速度低字节
[9]   BCC        校验 = XOR(byte[0]..byte[8])
[10]  0x7D       帧尾
```

速度编码示例：
- 前进 0.1 m/s：Vx=100 → `[0x00, 0x64]`
- 后退 0.2 m/s：Vx=-200 → `[0xFF, 0x38]`
- 停止：Vx=Vy=Vz=0

### 4.2 STM32 → A1 状态帧（24 字节）

```
字节      内容          说明
[0]       0x7B         帧头
[1]       Stop_Flag    停止标志
[2-3]     Vx           实际X速度
[4-5]     Vy           实际Y速度
[6-7]     Vz           实际Z角速度
[8-13]    Accel_X/Y/Z  加速度
[14-19]   Gyro_X/Y/Z   角速度
[20-21]   Voltage      电池电压 (mV)
[22]      BCC          校验 = XOR(byte[0]..byte[21])
[23]      0x7D         帧尾
```

## 七、新人必须理解的 4 个关键子系统

### 7.1 A1 板端主应用（`src/app_demo/`）

- **定位**：核心业务执行层
- **职责**：采集图像、跑人脸检测推理、UART 底盘控制、OSD 叠加、A1_TEST 协议处理
- **关键特性**：
  - 双推理后端：SCRFD（默认）/ YOLOv8，通过 project_paths.hpp 切换
  - Link-Test 模式：独立周期控制，与检测解耦
  - A1_TEST 协议：通过 COM13 远程调试
- **入门阅读**：
  1. `data/.../ssne_ai_demo/README.md`（板端应用文档）
  2. `demo_face.cpp`（主流程入口 + 键盘监听 + A1_TEST）
  3. `include/project_paths.hpp`（关键配置）

### 7.2 Aurora 工具链（`tools/aurora/`）

- **定位**：Windows 侧调试入口
- **功能**：
  - 摄像头预览（支持旋转、截图、画廊）
  - 双模式底盘调试：直连 STM32 / 经由 A1（A1_TEST）
  - A1 调试终端：内置 serial_terminal.py
- **启动**：
  ```powershell
  cd tools/aurora
  .\launch.ps1
  ```
- **阅读**：`tools/aurora/README.md`

### 7.3 STM32 底盘与双板通信

- **定位**：执行机构与底层运动控制层
- **职责**：接收上位控制指令并驱动电机/编码器闭环

## 九、新人常见误区

1. **一上来就改底层驱动**：建议先在上层验证链路，避免定位复杂问题
2. **不区分第三方代码与自研代码**：优先改自研目录，减少引入维护风险
3. **只看代码不看文档**：本仓库文档较完整，先读文档能节省大量试错时间
4. **首次构建后仍全量编译**：日常应优先使用增量脚本，提升迭代效率
5. **忽略双后端配置**：确认当前模型与后端匹配，避免推理失败

## 附录：A1_TEST 调试技巧

### 验证 A1 ↔ STM32 链路

1. 启动 A1 板端程序
2. 打开 Aurora Companion
3. 切换到底盘调试 → 经由 A1 模式
4. 连接 COM13
5. 发送 `A1_TEST test_echo hello`，确认有 JSON 回显
6. 发送 `A1_TEST link_test on`，观察底盘是否按周期运动
7. 发送 `A1_TEST stop`，观察底盘是否立即停止

### 调试 UTF-8 解码问题

如果终端出现乱码，可能是：
- UTF-8 字符被分包切分
- 使用了非 UTF-8 编码的中文
- 解决：参考 `tools/aurora/serial_terminal.py` 中的 `_normalize_text()` 函数
