# 15. AI 模型转换与部署

当前评委演示主线使用 `test.m1model` 5 分类视觉导航模型：板端读取 `person / stop / forward / obstacle / NoTarget`，并映射为 `forward -> forward`、其他 -> `stop`。本文后半部分仍保留 YOLOv8 转换说明，作为历史检测链路参考，不是当前默认上板演示路径。

## 当前默认部署模型

```text
/app_demo/app_assets/models/test.m1model
```

对应板端代码：

```cpp
array<int, 2> cls_shape = {320, 320};
string model_path = "/app_demo/app_assets/models/test.m1model";
```

Aurora 验证入口：`A1_TEST rps_snapshot <request_id>` 或页面内分类快照按钮。

## 一、RPS 分类模型部署流程

### 1. 模型准备

- 训练目标：`person / stop / forward / obstacle / NoTarget`（5 类）
- 输入尺寸：`320×320`（Y8 灰度，从 720×1280 摄像头帧中心裁剪）
- 板端前处理：与 `ssne_ai_demo` 内 `RPS_CLASSIFIER` 保持一致
- 分类输出：`forward -> forward`，其他 -> `stop`

### 2. 板端放置模型

将 `.m1model` 文件放置到：

```text
data/A1_SDK_SC132GS/smartsens_sdk/smart_software/src/app_demo/face_detection/ssne_ai_demo/app_assets/models/
```

当前默认目标模型名：

```text
test.m1model
```

### 3. 运行验证

- 板端运行 `cd /app_demo && ./scripts/run.sh`
- Aurora 页面访问 `http://127.0.0.1:6201`
- 调用 `A1_TEST rps_snapshot <request_id>` 核对 `label / confidence / scores / action`

## 二、YOLOv8 转换参考

以下内容仅用于保留历史检测模型转换方法。

### 1. 安装依赖

```bash
pip install ultralytics
```

### 2. 导出模型

```python
from ultralytics import YOLO

model = YOLO("yolov8n.pt")
model.export(format="onnx")
```
