# 贡献指南

## 改动前先判断改哪里
- 板端行为：先看 `data/.../ssne_ai_demo/`
- 镜像打包：先看 `scripts/` 和 SDK 打包路径
- Windows 工具：先看 `tools/aurora/`、`tools/PC/`、`tools/A1/`
- 文档：先看 `README.md`、三套工具 README 和 `docs/`

## 先分清一方代码和厂商内容
优先修改：
- `scripts/`
- `tools/aurora/`
- `tools/PC/`
- `tools/A1/`
- `docs/`
- `data/A1_SDK_SC132GS/smartsens_sdk/smart_software/src/app_demo/face_detection/ssne_ai_demo/`

谨慎修改：
- `data/A1_SDK_SC132GS/smartsens_sdk/` 其余部分
- `third_party/ultralytics/`
- `WHEELTEC_C50X_2025.12.26/`
- `output/`

## 提交前最小验证
- 文档：检查链接、命令、脚本名、端口、路径
- Windows 工具 Python：
  ```bash
  python -m py_compile tools/aurora/aurora_companion.py tools/aurora/serial_terminal.py tools/aurora/qt_camera_bridge.py tools/PC/pc_tool.py tools/PC/pc_chassis.py tools/A1/a1_tool.py tools/A1/a1_relay.py tools/A1/a1_serial.py
  ```
- 板端：优先做 `build_incremental.sh` 或 `build_complete_evb.sh`

## 文档改动要求
- 入口文档描述当前默认路径
- 专题文档只在确有对应代码时声明“已支持”
- 不要把未来规划、历史计划或猜测写成当前现状

## 构建和部署边界
- `build_complete_evb.sh` 对应完整镜像路径
- 最终部署关注 `zImage.smartsens-m1-evb`
- Windows 工具负责联调，不负责固件烧录本身

## 提交范围建议
- 尽量让一次提交围绕同一块功能或同一类文档问题
- 不要顺手混入无关改动
- 如果修改了 vendor-heavy 区域，说明原因并缩小改动面

## 遇到不确定时先查哪里
1. `README.md`
2. `CLAUDE.md`
3. 对应目录下的 README / 文档
4. 相关脚本和入口文件
