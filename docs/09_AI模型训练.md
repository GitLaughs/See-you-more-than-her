# 09. AI 模型训练

本文档说明当前仓库里用于 A1 评委演示的训练思路：以 demo-rps 风格的轻量分类为主，而不是继续走 YOLO 检测主线。

## 1. 训练目标

当前演示主线使用 RPS 分类测试：

- 输入原始采集图：`720×1280` 灰度图
- 板端裁剪：中心裁剪到 `720×540`
- 训练 / 部署输入：`320×320`
- 板端模型：`model_rps.m1model`
- 语义标签：`P / R / S / NoTarget`
- 动作映射：`P -> forward`，其他 -> `stop`

## 2. 数据集约定

训练和板端都必须共用同一套裁剪和类别顺序，避免训练/部署不一致。

推荐目录结构：

```text
dataset/
  raw/
    videos/
      P/
      R/
      S/
      NoTarget/
  processed/
    train/
    val/
    test/
```

## 3. 数据制作

当前分类数据制作思路是：

1. 从视频逐帧读取
2. 先做中心裁剪
3. 再做固定 ROI 裁剪
4. 按帧间隔抽样
5. 自动划分 `train / val / test`

对应脚本和说明放在 `demo-rps/dataprocess_modeltrain/`。

## 4. 训练模型

目标是单标签 4 类分类：

- `P`
- `R`
- `S`
- `NoTarget`

训练时要确认：

- 类别顺序和板端 `RPS_CLASSIFIER` 输出顺序一致
- `NoTarget` 作为正常类别参与训练，不做额外兜底
- 训练输出和板端读取方式一致

## 5. 导出与部署

训练完成后：

1. 导出 ONNX
2. 转换为 `.m1model`
3. 放到 `data/A1_SDK_SC132GS/smartsens_sdk/smart_software/src/app_demo/face_detection/ssne_ai_demo/app_assets/models/`
4. 板端默认指向 `model_rps.m1model`

## 6. 验证

当前评委演示验证链路是：

- Aurora 页面内调用 `/api/a1/rps_snapshot`
- 板端回传最新分类状态
- `A1_TEST rps_snapshot <request_id>` 可直接查看当前分类结果

## 7. 旧检测链路说明

仓库里保留少量 YOLO 相关历史文档，主要作为旧检测链路参考。当前默认路径不是它。

## 8. 常见问题

- 问题：训练和板端标签顺序不一致
  - 处理：检查数据集目录顺序、训练脚本 class list、板端输出映射

- 问题：板端输出全是 `NoTarget`
  - 处理：检查模型是否还是 `model_rps.m1model`，以及输入裁剪是否和训练一致

- 问题：还想继续做 YOLO
  - 处理：那是另一条主线，不是当前评委演示路径
