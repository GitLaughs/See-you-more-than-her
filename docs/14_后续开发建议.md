# 14 后续开发建议

这篇不是需求清单，而是对当前仓库最值得继续做的事情的建议。

## 1. 继续统一语义

- Windows 纯预览和 A1 板端预览保持明确区分
- 不要再让一个下拉框同时承担“源”和“模式”两种含义
- 文档里也尽量用同一套词汇

## 2. 强化板端 Demo

- 保持 `person` 触发逻辑稳定
- 把模型、阈值和 OSD 配置进一步参数化
- 如有需要，再抽出更清晰的配置层

## 3. 增强 Windows 工具

- 让串口扫描更容易看出新增设备
- 给 A1 Viewer 和 Companion 加更多状态提示
- 把常见操作做成更少的点击路径

## 4. ROS2 与雷达

- ROS2 优先做编排，不要重复实现底层协议
- 雷达等扩展能力建议放进独立节点
- 先保证主链路，再做扩展链路

## 5. 工程化建议

- 每次改动都跑一次静态检查
- 重要链路保持可复现的构建步骤
- 文档重写以后继续保持更新，不要再回到旧版口径
