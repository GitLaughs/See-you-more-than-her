# 后续开发建议

## 值得继续推进的方向

### ROS2 集成收敛
先明确哪些包属于默认路径，哪些继续保持 `COLCON_IGNORE`。这件事不解决，入口文档和真实工作区状态很容易继续漂。

### Aurora 联调体验
继续优化相机初始化、串口反馈和 UI 可观测性。Windows 侧联调已经能用，但稳定性和排障反馈还有继续收紧空间。

### 板端运行验证
围绕 `ssne_ai_demo`、A1_TEST 和 UART 控制建立更稳定的回归检查路径。这样每次改板端行为，不必重新发明验证方式。

### 镜像打包链路可见性
继续收敛 `build_complete_evb.sh`、`build_incremental.sh` 和最终产物之间的认知差。很多误解都来自“把 Demo 二进制当成最终交付物”。

## 不建议在入口层直接写成已完成的方向
- RPLidar 全量集成默认可用
- 深度感知 / 点云避障已经进入默认运行栈
- Nav2 / SLAM 已经是板端默认能力

这些更适合继续放在专题文档里，按代码实际落地情况逐步收敛。

## 建议工作方式
- 小步验证，先保证默认路径稳定
- 板端、Aurora、ROS2 三条链路分开描述，再谈集成
- 先修当前事实，再补未来路线
