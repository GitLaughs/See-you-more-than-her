# 常见问题

## `output/evb/` 没有新固件
确认执行的是 `build_complete_evb.sh`，而不是只做了 `build_incremental.sh`。

推荐直接看：

```bash
docker exec A1_Builder bash -lc "bash /app/scripts/build_complete_evb.sh"
```

最终关注产物：

```text
output/evb/<timestamp>/zImage.smartsens-m1-evb
```

## Aurora 页面打不开
确认 `tools/aurora/launch.ps1` 已启动，并访问 `http://127.0.0.1:6201`。

如果报 `WinError 10013` / “以一种访问权限不允许的方式做了一个访问套接字的尝试”，先检查 Windows TCP excluded port range。当前默认端口已避开本机保留段：Aurora `6201`、PC `6202`、A1 `6203`。

## Companion 没画面
当前已接受流程：先打开 `Aurora.exe` 完成相机初始化，再由 Companion 接管。

如果还不行，再试：
- `./launch.ps1 -Source a1`
- `./launch.ps1 -Source windows`
- `./launch.ps1 -SkipAurora`

## COM13 连接失败
确认：
- 串口号是不是 `COM13`
- 波特率是不是 `115200`
- A1 板端程序是否已经运行

## ROS2 包编不过
先确认包在 `src/ros2_ws/src/` 下，且没有 `COLCON_IGNORE`。

可以先做窄范围构建：

```bash
bash scripts/build_ros2_ws.sh <package-name>
```

## 只改了 `ssne_ai_demo`，为什么板上还是旧版本
因为增量构建不等于完整镜像重打包。只重编 Demo 后，如果你要得到新的可烧录镜像，还需要重新跑 `build_complete_evb.sh`。

## `data/A1_SDK_SC132GS/` 不存在
说明 SmartSens SDK 还没准备好。优先重新跑：

```bash
bash scripts/bootstrap.sh
```

如果本机没有基础镜像，还要加：

```bash
bash scripts/bootstrap.sh --load-image /path/to/a1-sdk-builder-latest.tar
```

## `a1-sdk-builder:latest` 不存在
说明基础镜像未加载或最终镜像未构建。先检查 tar 包路径，再重跑 bootstrap，或手动执行：

```bash
docker load -i /path/to/a1-sdk-builder-latest.tar
docker build -f docker/Dockerfile -t a1-sdk-builder:latest .
```

## 板端程序能跑，但 Windows 工具控不了底盘
先区分你走的是哪条链路：
- 直连 STM32：使用 `tools/PC/launch.ps1`，默认 `http://127.0.0.1:6202`
- 经由 A1 的 COM13 / A1_TEST：使用 `tools/A1/launch.ps1`，默认 `http://127.0.0.1:6203`

两条链路排查点不同，不要混在一起看。

## 文档里写的路径和文件名对不上
以根 `README.md` 和当前编号文档为准。旧文件名、旧端口、旧链接已经不再是入口事实来源。
