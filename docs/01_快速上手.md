# 01 快速上手

## 本文适合谁
第一次接触仓库、想先把环境起起来并跑通一次完整流程的同学。

## 你会接触到的 4 块内容
- 板端 Demo：`data/A1_SDK_SC132GS/smartsens_sdk/smart_software/src/app_demo/face_detection/ssne_ai_demo/`
- SDK / 固件打包：`data/A1_SDK_SC132GS/smartsens_sdk/`
- ROS2：`src/ros2_ws/`
- Windows 工具：`tools/aurora/`、`tools/PC/`、`tools/A1/`

## 最短路径
1. `bash scripts/bootstrap.sh`
2. `docker compose -f docker/docker-compose.yml up -d`
3. `docker exec A1_Builder bash -lc "bash /app/scripts/build_complete_evb.sh --skip-ros"`
4. `ssh root@<A1_IP>` 后运行 `/app_demo/scripts/run.sh`
5. Windows 侧执行 `tools/aurora/launch.ps1`

默认地址：
- Aurora 视频 / COM13 终端：`http://127.0.0.1:6201`
- PC 直连 STM32：`http://127.0.0.1:6202`
- A1 中继控制：`http://127.0.0.1:6203`

## 先理解什么是默认路径

### 板端默认运行栈
- 默认上板运行路径是 `ssne_ai_demo`
- 图像采集、SCRFD/YOLOv8 推理、OSD、A1_TEST、UART 底盘控制都在这条路径上
- 直接上板验证时，优先看 `/app_demo/scripts/run.sh`

### ROS2 不是默认板端运行栈
- `src/ros2_ws/` 是独立工作区和后续集成路径
- `scripts/build_ros2_ws.sh` 只扫描 `src/ros2_ws/src/` 下的包
- 一些可选包故意保留 `COLCON_IGNORE`

### Windows 侧工具拆分
- `tools/aurora/`：相机预览、拍照、COM13 终端、A1_TEST 手动测试
- `tools/PC/`：电脑直连 STM32 调试
- `tools/A1/`：COM13 → A1_TEST → STM32 中继控制
- 当前接受的 Aurora 相机初始化流程：先打开 `Aurora.exe`，再由 Companion 接管

## 第一次改代码时怎么找入口
- 改板端行为：先看 `ssne_ai_demo/`
- 改镜像打包：先看 `scripts/` 和 `data/A1_SDK_SC132GS/smartsens_sdk/`
- 改 Aurora：先看 `tools/aurora/`
- 改 ROS2：先看 `src/ros2_ws/`

## 第一次验证建议
- 只改文档：检查链接、命令、脚本名、端口
- 改 Aurora：至少跑一遍 Python 语法检查
- 改 ROS2：优先做包级构建
- 改板端：优先走 `build_incremental.sh` 或 `build_complete_evb.sh`
