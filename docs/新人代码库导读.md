# 新人代码库导读

本文档面向第一次接触本仓库的同学，给出“先看什么、再做什么、怎么逐步深入”的路线。

## 1. 代码库整体结构（先建立地图）

```text
See-you-more-than-her/
├── README.md                       # 项目总览：功能、目录、快速开始
├── docs/                           # 文档中心（建议按顺序阅读）
├── scripts/                        # 构建脚本（全量/增量/ROS2）
├── src/
```

## 2. 新人必须理解的 4 个关键子系统

### 2.1 A1 板端主应用（`src/a1_ssne_ai_demo/`）
- 定位：**核心业务执行层**。
- 主要职责：采集图像、跑人脸/YOLOv8 推理、处理激光雷达、OSD 叠加、通过 TCP 向 Aurora 输出调试数据。
- 入门阅读建议：
  1. `src/a1_ssne_ai_demo/README.md`
  2. `demo_vision.cpp`（主流程入口）
  3. `include/project_paths.hpp`（路径、阈值、端口等关键配置）

### 2.2 ROS2 工作区（`src/ros2_ws/`）
- 定位：**机器人系统集成层**。
- 主要职责：底盘通信、传感器节点、导航与启动编排。
- 新人最需要先知道：
  - `turn_on_wheeltec_robot/launch/`：常用 launch 启动脚本。
  - `wheeltec_robot_msg/`：消息定义（理解话题输入输出的基础）。
  - `serial/`、`web_video_server-ros2/`、`aruco_ros-humble-devel/` 包含较多第三方或移植代码，阅读时先“会用”再“深挖”。

### 2.3 STM32 底盘与双板通信（`WHEELTEC_C50X_2025.12.26/` + `src/stm32_akm_driver/`）
- 定位：**执行机构与底层运动控制层**。
- 主要职责：接收上位控制指令并驱动电机/编码器闭环。
- 新人建议：先阅读 `src/stm32_akm_driver/README.md` 理解协议，再决定是否进入 `WHEELTEC_C50X_2025.12.26/` 深入到 FreeRTOS 与硬件驱动实现。

### 2.4 工程化与工具链（`scripts/` + `docker/` + `tools/`）
- 定位：**开发效率层**。
- 主要职责：统一构建环境、固件产物收集、训练与调试工具支持。
- 新人建议优先掌握：
  - `scripts/build_src_all.sh`（首次全量构建）
  - `scripts/build_incremental.sh`（日常增量开发）
  - `tools/aurora/`（快速验证算法输出和链路联通）

## 3. 推荐阅读顺序（1~2 天起步）

### Day 1：跑通与理解全景
1. `README.md`（15-20 分钟）：了解项目边界和目标。
2. `docs/快速上手指南.md`（30-45 分钟）：按步骤启动容器并完成一次构建。
3. `docs/双板架构设计.md`（20 分钟）：建立 A1 ↔ STM32 的通信认知。
4. `docs/编译手册.md`（按需）：对照脚本验证自己的构建命令。

### Day 2：进入代码
1. `src/a1_ssne_ai_demo/README.md`：建立主程序模块图。
2. 从入口文件开始顺藤摸瓜（`demo_vision.cpp` → 各模块 `.cpp/.hpp`）。
3. 查看 `src/ros2_ws/README.md` 与关键 launch，理解系统如何被一键拉起。
4. 用 Aurora 观察实际输出（点云、检测框、障碍区域），把“代码逻辑”与“运行现象”对齐。

## 4. 新人常见误区

1. **一上来就改底层驱动**：建议先在上层验证链路，避免定位复杂问题。
2. **不区分第三方代码与自研代码**：优先改自研目录，减少引入维护风险。
3. **只看代码不看文档**：本仓库文档较完整，先读文档能节省大量试错时间。
4. **首次构建后仍全量编译**：日常应优先使用增量脚本，提升迭代效率。

## 5. 后续学习建议（按角色）

### 视觉算法方向
- 深入 `yolov8_detector.*`、`project_flow.*`、`osd_visualizer.*`。
- 配合 `docs/YOLOv8训练指南.md` 走完“训练→转换→部署→验证”闭环。

### 机器人系统方向
- 深入 `src/ros2_ws/src/` 中底盘/启动/消息包。
- 重点理解 launch 依赖关系、话题命名和参数文件。

### 嵌入式底盘方向
- 深入 STM32 工程中的 UART、编码器、电机控制和任务调度。
- 建议建立“协议抓包 + 实车动作”对照记录，提升排障效率。

## 6. 一个可执行的 2 周上手目标

- 第 1 周：独立完成环境搭建、全量构建、增量构建、Demo 运行与日志采集。
- 第 2 周：选择一个小模块（例如阈值参数或调试字段）完成一次“小改动→编译→运行→验证→提交”。

达到这个目标后，再进入跨模块优化（例如视觉检测结果影响避障策略），学习效率会更高。
