# 02 环境搭建

## 本文定位
这份文档只讲当前仓库的推荐初始化路径：先用 `bootstrap.sh` 把 SDK、镜像和容器准备好；手动步骤只在脚本不适用时兜底。

## 前提条件
- Git
- Docker / Docker Desktop
- 足够的磁盘空间用于 SDK、镜像和构建缓存
- 可以获取 SmartSens SDK 和基础镜像

## 推荐方式：使用 bootstrap 脚本

```bash
bash scripts/bootstrap.sh --load-image /path/to/a1-sdk-builder-latest.tar
```

如果基础镜像已经在本机 Docker 中存在：

```bash
bash scripts/bootstrap.sh
```

脚本会负责：
- 检查并拉取 `data/A1_SDK_SC132GS/`
- 可选加载基础 Docker 镜像
- 构建最终 `a1-sdk-builder:latest`
- 启动 `A1_Builder` 容器

## 初始化完成后检查什么

```bash
docker ps --filter name=A1_Builder
docker exec A1_Builder bash -lc "ls /app/data/A1_SDK_SC132GS/smartsens_sdk/smartsens_sdk/scripts/"
docker exec A1_Builder bash -lc "test -f /app/scripts/build_complete_evb.sh"
```

预期：
- `A1_Builder` 容器在运行
- SDK 路径存在
- 容器内可用项目构建所需基础环境

## 手动方式：只在脚本不适用时使用

### 1. 拉取 SDK

```bash
git clone --depth 1 https://git.smartsenstech.ai/Smartsens/A1_SDK_SC132GS.git data/A1_SDK_SC132GS
```

### 2. 加载基础镜像

```bash
docker load -i /path/to/a1-sdk-builder-latest.tar
```

### 3. 构建最终镜像并启动容器

```bash
docker build -f docker/Dockerfile -t a1-sdk-builder:latest .
docker compose -f docker/docker-compose.yml up -d
```

## 目录边界提醒
- `data/A1_SDK_SC132GS/smartsens_sdk/` 是 SDK / vendor-heavy 区域，谨慎修改
- 日常集成工作优先落在 `scripts/`、`tools/aurora/`、`docs/`、`ssne_ai_demo/`

## 常见初始化问题

### `data/A1_SDK_SC132GS/` 不存在
说明 SDK 没拉下来。优先重新运行 `bash scripts/bootstrap.sh`，再看是否需要手动克隆。

### `a1-sdk-builder:latest` 不存在
说明基础镜像未加载或最终镜像未构建。先确认 `--load-image` 是否传入正确 tar 包，再重跑 bootstrap。

### 容器没起来
先看：

```bash
docker compose -f docker/docker-compose.yml ps
docker compose -f docker/docker-compose.yml logs
```
