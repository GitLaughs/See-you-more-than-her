# 接口接线说明 — A1 开发板 ↔ STM32 底盘

## 一、概述

本文档说明 SmartSens A1 开发板与 WHEELTEC C50X 底盘（STM32F407）之间的物理接线方法。A1 通过 GPIO UART0 接口向 STM32 的 UART3 发送运动控制指令，实现人脸检测驱动小车。

## 二、必须连接的线路

### 2.1 接线总表

| 序号 | A1 开发板引脚 | 功能 | 方向 | STM32 引脚 | 功能 | 线材颜色建议 |
|------|--------------|------|------|-----------|------|-----------|
| 1 | P4-15 (GPIO_PIN_0) | UART TX0 | A1 → STM32 | PB11 | UART3 RX | 绿色 |
| 2 | P4-16 (GPIO_PIN_2) | UART RX0 | STM32 → A1 | PB10 | UART3 TX | 黄色 |
| 3 | P4-33 或 P4-34 | GND | 双向 | GND | GND | 黑色 |

> **重要**: GND（地线）是 UART 通信的必要条件，必须连接！

### 2.2 接线示意图

```
    A1 开发板 (P4 接口)                WHEELTEC C50X (STM32F407)
    ┌─────────────────┐                ┌─────────────────────┐
    │                 │                │                     │
    │  P4-15 (TX0)  ──┤── 绿线 ──────→├── PB11 (UART3_RX)  │
    │  GPIO_PIN_0     │                │                     │
    │                 │                │                     │
    │  P4-16 (RX0)  ──┤── 黄线 ──────←├── PB10 (UART3_TX)  │
    │  GPIO_PIN_2     │                │                     │
    │                 │                │                     │
    │  P4-33 (GND)  ──┤── 黑线 ──────→├── GND               │
    │                 │                │                     │
    └─────────────────┘                └─────────────────────┘
```

## 三、A1 开发板引脚详解

### 3.1 P4 接口 (ZX-PZ2.54-2-25PZZ)

A1 开发板的 P4 接口是一个 50 针双排排针（2.54mm 间距）。

**UART 相关引脚:**

| P4 引脚号 | GPIO 编号 | 默认功能 | 电平 | 说明 |
|-----------|----------|---------|------|------|
| 15 | GPIO_PIN_0 (D0) | **UART TX0** | 1.8V | 默认配置为 UART 发送 |
| 16 | GPIO_PIN_2 (D2) | **UART RX0** | 1.8V | 默认配置为 UART 接收 |

**电源与地引脚:**

| P4 引脚号 | 功能 | 说明 |
|-----------|------|------|
| 33, 34, 47, 48 | GND | 接地（任选一个） |
| 35, 36 | 1.8V | UART 信号电平（仅参考，无需外接） |

### 3.2 GPIO 可用引脚 (供扩展使用)

| GPIO 引脚 | P4 引脚号 | 默认功能 | 可用场景 |
|-----------|----------|---------|---------|
| GPIO_PIN_0 | 15 | UART TX0 | **已占用 (底盘通信)** |
| GPIO_PIN_2 | 16 | UART RX0 | **已占用 (底盘通信)** |
| GPIO_PIN_8 | 17 | HREF | 通用 GPIO 输入/输出 |
| GPIO_PIN_9 | 18 | VS | 通用 GPIO 输入/输出 |
| GPIO_PIN_10 | 19 | PCLK | 通用 GPIO 输入/输出 |

> GPIO_PIN_1, 3, 4, 5, 6, 7 已被系统占用，不可使用。

## 四、STM32 端引脚详解

### 4.1 UART3 (用于 A1 通信)

| STM32 引脚 | 功能 | 说明 |
|-----------|------|------|
| PB10 | UART3_TX | 向 A1 发送状态数据 (24字节帧) |
| PB11 | UART3_RX | 接收 A1 控制指令 (11字节帧) |

**UART3 配置参数:**
- 波特率: 115200
- 数据位: 8
- 停止位: 1
- 校验: 无
- 中断优先级: 0 (最高，确保数据不丢失)

### 4.2 其他 UART 接口 (参考)

| UART | 引脚 | 波特率 | 用途 | 是否被占用 |
|------|------|--------|------|-----------|
| UART1 | PA9/PA10 | 115200 | 调试/手柄接收 | 可共用 |
| **UART3** | **PB10/PB11** | **115200** | **A1 控制** | **本项目占用** |
| UART4 | PC10/PC11 | 115200 | OLED/蓝牙APP | 可共用 |

## 五、电平兼容性注意事项

### 5.1 电平说明

| 设备 | IO 电平 | 说明 |
|------|---------|------|
| A1 开发板 | **1.8V** | GPIO 输出电平为 1.8V |
| STM32F407 | **3.3V** | 标准 3.3V 逻辑电平 |

### 5.2 兼容性分析

- **A1 TX → STM32 RX (1.8V → 3.3V 接收)**:
  STM32F407 的输入高电平阈值约为 VDD×0.42 = 1.39V。
  A1 输出 1.8V > 1.39V，**可以直接连接**。

- **STM32 TX → A1 RX (3.3V → 1.8V 接收)**:
  ⚠️ STM32 输出 3.3V 高电平，超过 A1 的 1.8V IO 电压。
  **建议使用电平转换器** 或分压电路（如 1kΩ + 1.8kΩ 分压）。
  若 A1 的 GPIO 引脚具有 3.3V 容忍能力，可直接连接（需确认）。

### 5.3 简易分压电路 (STM32 TX → A1 RX)

```
STM32 PB10 (TX) ──── R1 (1kΩ) ──┬── A1 P4-16 (RX0)
                                  │
                            R2 (1.8kΩ)
                                  │
                                 GND

Vout = 3.3V × 1.8/(1.0+1.8) ≈ 2.12V → 仍略高于1.8V
可调整为: R1=2.2kΩ, R2=2.2kΩ → Vout ≈ 1.65V
```

> **测试阶段建议**: 先只连接 TX 线（A1→STM32 方向），验证控制指令。
> 状态反馈（STM32→A1）可在确认电平兼容性后再连接。

## 六、接线步骤

### 步骤 1: 准备工具
- 杜邦线 3 根 (公对母 或 公对公，取决于接口类型)
- 万用表 (可选，用于验证连接)
- 电平转换模块 (可选，用于 STM32→A1 方向)

### 步骤 2: 断电接线
1. **确保两块板子都已断电！**
2. 连接 GND: A1 P4-33 ← 黑线 → STM32 GND
3. 连接 TX: A1 P4-15 (TX0) ← 绿线 → STM32 PB11 (RX)
4. 连接 RX: A1 P4-16 (RX0) ← 黄线 → STM32 PB10 (TX)
   （如需电平转换，在此线路中间接入转换模块）

### 步骤 3: 上电验证
1. 先给 STM32 底盘上电
2. 再给 A1 开发板上电
3. 在 A1 运行程序:
   ```bash
   cd /app_demo
   ./run.sh
   ```
4. 观察终端输出:
   - `[底盘] UART 已初始化` — UART 初始化成功
   - `[DRIVE] 人脸检测到` — 检测到人脸，小车应前进
   - `[STOP] 未检测到人脸` — 未检测到人脸，小车应停止

### 步骤 4: 故障排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| UART 初始化失败 | 内核模块未加载 | `insmod gpio_kmod.ko` + `insmod uart_kmod.ko` |
| 小车无反应 | TX/RX 接反 | 互换绿线和黄线 |
| 小车无反应 | GND 未连接 | 检查地线 |
| 小车无反应 | 波特率不匹配 | 确认两端都是 115200 |
| 小车抖动 | 信号干扰 | 缩短线缆长度，远离电机 |
| 程序报串口错误 | 设备文件缺失 | 检查 `/dev/` 下 UART 设备 |

## 七、通信协议速查

### A1 → STM32 (11 字节控制帧)

```
[0x7B] [Cmd] [0x00] [Vx_H] [Vx_L] [Vy_H] [Vy_L] [Vz_H] [Vz_L] [BCC] [0x7D]
  │      │            │                                             │
  帧头   命令          速度数据 (int16_t, mm/s)                       帧尾
```

- Cmd=0x00: 正常控制
- BCC = byte[0] XOR byte[1] XOR ... XOR byte[8]
- 速度: int16_t 有符号, 单位 mm/s

### STM32 → A1 (24 字节状态帧)

```
[0x7B] [Flag] [Vx 2B] [Vy 2B] [Vz 2B] [Accel 6B] [Gyro 6B] [Volt 2B] [BCC] [0x7D]
```
