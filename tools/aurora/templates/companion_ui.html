电脑直连工作流

电脑直连 STM32

这一页只处理本机串口直连、链路验证与 ROS 直连调试。视频预览、模型切换和采集始终固定在左侧，不需要来回切页找入口。

1. 连接串口 选择本地 USB 串口并建立直连，默认波特率 115200。
2. 验证链路 在下方共享操作区执行联通测试，确认控制下发和遥测回传都正常。
3. 进入联调 确认遥测正常后，再使用运动控制、原始帧发送或 ROS 节点联调。
PC端实时 ONNX 模型切换
模型即时切换 (onnx/pt)。推理基于当前摄像头灰度帧，统一走 720×540 中心裁剪后缩放到 640×480 的 A1 主链。
串口连接与协议
协议参数
帧头: 0x7B   帧尾: 0x7D
指令帧: 11 字节 → STM32
遥测帧: 24 字节 ← STM32
校验: XOR(bytes[0..N-2])
如果后续要让 ROS 节点占用同一串口，请先在这里断开直连连接，避免串口竞争。
直连前检查
1. 电脑 USB 串口连接 STM32 UART3，并确保共地。
2. 在左侧确认视频源正常，在本页完成串口连接。
3. 到下方共享操作区执行联通测试，发送零速度 11 字节指令帧。
4. 收到 24 字节遥测后，再进行运动控制或 ROS 联调。
联通测试发送帧格式：7B 00 00 00 00 00 00 00 00 BCC 7D。若仅显示“单向连通”，通常是 STM32 回传线未接通。

COM13 工作流

经由 A1 到 STM32

这一页聚焦 PC → COM13 → A1_TEST → STM32 的本地串口链路。所有经由 A1 的测试和底盘控制都统一走 COM13 终端。

1. 连接 COM13 优先自动连接 USB-HiSpeed-SERIAL-A / CH347F 对应的 COM13。
2. 发送 A1_TEST 通过 A1 demo 内置 CLI 下发 debug_status、move、stop 等命令。
3. 在共享区控制 联通测试、运动控制和日志会自动改为 COM13 目标。
COM13 / A1_TEST 连接
COM13 状态：未检测
命令通道：A1_TEST
左侧预览建议切到 A1 板端 OSD；底盘命令会通过 COM13 发送到 A1 demo，再由 A1 UART0 控制 STM32。
经由 A1 的工作顺序
1. 点击“连接 COM13”，确认 A1 demo 的串口终端在线。
2. 到下方共享操作区执行联通测试，发送 A1_TEST debug_status。
3. 使用运动控制时会发送 A1_TEST move vx vy vz，松开或急停会发送 A1_TEST stop。
4. 如需看详细回传，直接查看 A1 串口终端实时输出。
接线参考
COM13 控制场景下，PC 通过 A1 串口终端下发 A1_TEST，A1 通过 UART0 与 STM32 UART3 直连并共地。
场景发送端 (TX)→接收端 (RX)说明
A1→STM GPIO_PIN_0 (UART0 TX) → PB11 (UART3 RX) A1 控制 STM32
STM→A1 PB10 (UART3 TX) → GPIO_PIN_2 (UART0 RX) 遥测回传
共地 A1 GND STM32 GND 必须共地
PC TX USB串口 TX → PB11 (UART3 RX) PC 调试用
PC RX PB10 (UART3 TX) → USB串口 RX PC 接收遥测
A1 GPIO UART 为 3.3V TTL 电平；STM32 UART3 同为 3.3V，可直连，勿接 5V
指令帧结构 (11字节: A1→STM32)
7B[0]帧头
Cmd[1]命令
00[2]保留
Vx_H[3]
Vx_L[4]
Vy_H[5]
Vy_L[6]
Vz_H[7]
Vz_L[8]
BCC[9]
7D[10]帧尾
Cmd: 0x00=运动 · 0x01/0x02=回充 · 0x03=对接  |  速度单位: mm/s (int16 大端)
