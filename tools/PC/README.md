# PC Windows 工具

PC 工具是 Windows 侧电脑直连 STM32 调试入口，不包含摄像头、COM13 终端、A1 中继控制或 ROS 调试。

## 适用场景
- 电脑 USB 串口直连 STM32 UART
- 直连链路联通测试
- 前后/左右/停止控制
- STM32 遥测查看
- TX/RX 通信帧查看

## 启动

```powershell
cd tools/PC
.\launch.ps1
```

默认地址：`http://127.0.0.1:6202`

默认通信端口：`COM17`

如需改 Web 端口：

```powershell
.\launch.ps1 -Port 6212
```

## 安全连接行为
- 连接或切换端口时，页面会进入连接中状态并禁用连接相关按钮。
- 串口打开后会立即发送一次零速度停止帧。
- 断开串口前也会先发送一次零速度停止帧。
- 页面只有在串口打开且安全停止帧发送成功后才显示已连接。

## 通信日志
- TX 表示 PC 发给 STM32 的控制帧。
- RX 表示 STM32 返回给 PC 的遥测帧。
- 日志区域固定高度，内部滚动，不会一直撑高页面。
- 每条日志显示原始 HEX、方向、字段和含义。

## 边界
- 不启动 Qt camera bridge
- 不探测摄像头
- 不使用 COM13 / A1_TEST 终端
- 不提供 A1 中继控制
- 不提供 ROS 状态或节点调试

## 常见问题

### 默认端口为什么不是 5802？
Windows 当前会保留 `5730-5929` 端口段，`5802` 会触发 `WinError 10013` / “以一种访问权限不允许的方式做了一个访问套接字的尝试”。因此默认改为 `6202`。

### 一插串口小车就动
先确认选中端口是直连 STM32 的端口，默认优先使用 `COM17`。PC 工具连接成功前会发送停止帧；如果仍然运动，优先检查 USB 串口线、STM32 UART 接线、DTR/RTS 跳线和底盘固件对串口上电噪声的处理。
